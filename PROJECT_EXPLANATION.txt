WHAT THIS PROJECT DOES:
-----------------------
This project analyzes Twitter sentiment in real-time using Big Data 
technologies. It reads tweets, determines if they're positive, negative, 
or neutral, and displays the results on a live dashboard.


HOW IT WORKS (3 SIMPLE STEPS):
-------------------------------

STEP 1: Tweet Producer
    - Reads the Sentiment140 dataset (1.6 million real tweets, 228MB)
    - Sends tweets one-by-one through a socket connection (port 9999)
    - Simulates a real-time Twitter stream

STEP 2: Spark Processor
    - Apache Spark listens to the socket and receives tweets
    - Uses TextBlob library to analyze each tweet's sentiment:
        * Positive: "I love this!" → Polarity > 0
        * Negative: "This is terrible" → Polarity < 0
        * Neutral: "It is what it is" → Polarity = 0
    - Saves results to a CSV file (data/tweets.csv)

STEP 3: Dashboard
    - Streamlit web app reads the CSV file every 2 seconds
    - Displays:
        * Total tweet count
        * Positive/Negative/Neutral counts
        * Bar chart showing sentiment distribution
        * Table of recent tweets


KEY TECHNOLOGIES USED:
----------------------
1. PySpark - Big Data processing framework (handles massive datasets)
2. TextBlob - Natural Language Processing for sentiment analysis
3. Streamlit - Interactive web dashboard
4. Sentiment140 Dataset - Real-world Twitter data from Stanford
